# １．本報告の背景と目的 {#１．本報告の背景と目的}

## 1.1 背景 {#1.1-背景}

近年、GIGA スクール構想により全国の学校で 1 人 1 台端末の整備が急速に進み、教育現場におけるデジタル化が大きく前進している。これに伴い、学習活動の履歴情報であるスタディ・ログが多様な教育アプリケーションや学習管理システムから得られるようになり、学習状況の可視化、個別最適な学びの実現、教育改善に向けたエビデンス創出などへの活用可能性が高まっている。従来は取得が困難であった細粒度の学習データが、現在では初等中等教育段階においても収集・分析できる環境が整いつつあり、スタディ・ログの利活用は教育 DX を推進する基盤としてますます重要性を増している。

こうした中、スタディ・ログの技術標準として、米国ADLが策定し大学・企業・軍で広く採用されている xAPI（Experience API） や、米国1EdTech が策定した Caliper Analytics が注目されている。xAPI は、学習者の行動を「Actor（行為主体）」「Verb（行為）」「Object（対象）」といった構造で表すことで、学内外の多様な学習活動を統一的に記述できる枠組みを提供する。Caliper も xAPI 同様に学習活動を Actor, Verb, Object の組として記述し、LMS や教育アプリケーション間でデータを交換するための共通モデルを提供している。これらの標準は、異なる事業者が出力するスタディ・ログ間でのデータ互換性を確保し、学習分析の効率性を高める効果があることから、普及が進んでいる。

日本国内においては、2016年度に京都大学 緒方研究室が開始した科学研究費補助金(S)の事業においてスタディ・ログ記述にxAPIを採用し、また2018年度に開始した内閣府 SIP事業においてxAPI Profile を規定しており、日本におけるスタディ・ログはxAPIを用いて記述する流れが定着した。一方、スタディ・ログに含めるべき内容に関する議論や合意形成が事業者間でなされていないのが現状である。xAPIはスタディ・ログを記述する枠組みのみを提供しており、含めるべきデータの詳細は事業者に一任されている。この状況のままスタディ・ログ活用の社会実装を進めると、複数事業者が異なる記述内容や粒度のログを出力することになる。自治体や学校がこのログを統合して分析すると、その工数が増大し、またデータ処理の専門性が求められることで、現場でのログの利活用が阻害される恐れがある。また、教育委員会や学校にとってシステム選定時の比較可能性を低下させ、ログの蓄積や継続的活用の妨げにもなり得る。

そのため、日本の教育現場に適合する形でスタディ・ログの記述内容を整理し、事業者間で合意形成された統一的な記述形式を策定することが必要である。国際標準である xAPI の枠組みを参照しつつ、日本の学習指導要領やカリキュラム構造、学校現場で利用されているツールなどを踏まえた国内仕様を議論することが重要である。特に xAPI では、学習活動の種類ごとに記述ルールをまとめた「xAPI Profile」を定義できるため、日本の初等中等教育に適した Profile を策定することで、国際標準との互換性を保ちながら国内の実情に応じたデータ利活用を実現できる。

本報告はこうした背景を踏まえ、xAPIを用いて日本国内の事業者が共通して採用可能なスタディ・ログ記述形式を規定することで、教育データの円滑な流通と効果的な学習分析の実現を目指すものである。標準化された記述形式が普及することで、教育委員会や学校がスタディ・ログをより容易に活用でき、学習者一人ひとりに応じた教育支援の高度化や、教育政策のエビデンス基盤の強化につながることが期待される。＜タイトルにある効果を回収する必要＞

## 1.2 目的 {#1.2-目的}

本報告は上記の背景を踏まえ、日本における教育DX事業者が参加して策定した以下のxAPI Profile仕様を示すことを目的とする。

- 日本国内で流通するスタディ・ログが準拠すべき共通仕様 (JCore Profile)
- LMS (学習eポータル）が出力するスタディ・ログが準拠すべき仕様 (xx Profile)
- CBTが出力するスタディ・ログが準拠すべき仕様 (xx Profile)
- デジタル教科書が出力するスタディ・ログが準拠すべき仕様 (xx Profile)
- 学習支援ツールが出力するスタディ・ログが準拠すべき仕様 (xx Profile)

## 1.3 従来の取り組みと本報告の位置付け {#1.3-従来の取り組みと本報告の位置付け}

図1-1 に本報告書の内容と既存の xAPI Profile群やスタディ・ログの関係を示す。

図1-1 本報告書の内容と 既存の xAPI Profile群やスタディ・ログの関係

xAPI は、アメリカADL (Advanced Distributed Learning) が策定したスタディ・ログの記述仕様であり、以下の特徴を持つ。

- スタディ・ログをJSON形式で記述する
- スタディ・ログは Actor(主体)、Verb(動作)、Object(対象) などの情報を含む

一方、学習活動には様々な種類があり、その種類ごとにスタディ・ログに含むべき内容が異なる。例えば、デジタル教科書では「ページを見た」「アンダーラインを引いた」など、CBTでは「クイズを見た」「解答した」など、種類毎にVerb (動作) が異なる。 このため、学習活動ごとにスタディ・ログで記述すべき動詞、含めるべき情報、スタディ・ログの雛形（テンプレート）を定める効果が議論された。こういった情報を集約したのが xAPI Profile である。xAPI Profile は学習活動ごとに規定され、「どのようにProfileを書くべきか」という仕様が xAPI Profile Specification として規定されている（図1-1 最上部）。この Specification に準拠する形で、主にアメリカの団体や事業者が様々な xAPI Profile を策定・公開している（図1-1 左）。実際に蓄積されているxAPI スタディ・ログの件数に関する統計は存在しないが、オーストラリア Articulate社が開発したツール StoryLine の使用に伴って収集された xAPIスタディ・ログは月間180万件に及んだと紹介されている[^1]。  
　一方日本においては、事業者がxAPIスタディ・ログを実際に出力している例は稀である。京都大学 緒方研究室が開発したLEAFシステム（教材配信システム BookRollや解析ツール LogPaletteを含む）が京都や仙台にある複数の学校に導入され、実際の授業活動における xAPI 形式のスタディ・ログを出力している。＜xAPI Log を出力する他の例を紹介＞
